# rho-Calculus (Lean 4)

## What this is about

The **rho-calculus** (Meredith & Radestock 2005) is a concurrent process calculus
built on one striking move: *names are quoted processes.* In most calculi a channel
name is an atom drawn from some infinite alphabet; here every name `x` has the form
`@P` for an actual process `P`, and the governing equation `@(*(n)) = n` says
quoting and dereferencing are inverse. There is no separate supply of names — the
theory of names is *generated by* the theory of processes. That self-reference
(reflection) is exactly what lets a process build, communicate, and later run
references to code, which is why the rho-calculus underlies Mettapedia's
reflective-concurrency story.

This directory formalizes the calculus in Lean using a *locally nameless*
representation (de Bruijn indices for bound names, so alpha-equivalence is free),
proves an executable rewrite engine sound against the reduction relation, and then
develops two extensions: a **spice calculus** that gives agents an n-step lookahead
("past / present / future"), and a **rhometta** reduction layer that defers
MeTTa-style rewriting at the COMM point without forking the base reducer.

Part of `../../../../../../papers/process-calculi.tex` (Section 2).

## Syntax (locally nameless)

```
Pattern ::= pNil | pPar P Q | pInput x y P | pOutput x y | pDrop x | pQuote P
```

`pQuote`/`pDrop` implement the reflection. `pInput`/`pOutput` are the
communication primitives. Locally nameless representation throughout.

## Key Results

- **Executable engine sound** w.r.t. the reduction relation (`Engine.lean`,
  `reduceStep_sound`).
- **Spice calculus recovers standard rho-calculus at n = 0** (`SpiceRule.lean`,
  `futureStates_zero` / `reachableStates_zero`).
- **Reduction-defined modal Galois connection** (`Reduction.lean`,
  `galois_connection`): the "possibly" operator `◇` is left-adjoint to the "rely"
  operator `⧫`, both defined directly from the reduction relation. (This is the
  adjunction itself; it is not a full modal mu-calculus development — the broader
  Hennessy-Milner / modal-logic theory lives under `Mettapedia/OSLF/` and
  `Mettapedia/GSLT/`.)
- **Serializability of disjoint COMMs** (`ParallelWave.lean`).
