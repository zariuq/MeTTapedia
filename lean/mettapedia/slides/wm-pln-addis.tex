%% SLIDE 2: PLN already does 2D — but it's not enough
%% ══════════════════════════════════════════════════════
\begin{frame}{PLN's $(s, c)$ Pairs: What They Get Right and Wrong}

PLN already stores 2D truth values: $s = n^+/(n^+ + n^-)$,
$c = (n^+ + n^-)/(n^+ + n^- + k)$.

\medskip
\textbf{What this gets right}:
\begin{itemize}
\item For binary evidence, PLN's revision via $w = ck/(1{-}c)$
      IS addition of counts --- mathematically exact
\item Confidence grows with evidence; $(s, c)$ distinguishes
      $\langle 1, 1 \rangle$ from $\langle 100, 100 \rangle$
\end{itemize}

\medskip
\textbf{Where it breaks down}:
\begin{itemize}
\item \alert{Beyond binary}: for Dirichlet $(\alpha_1, \ldots,
      \alpha_k)$ or Normal-Gamma $(n, \sum x, \sum x^2)$,
      $(s, c)$ collapses $k$ or 3 dimensions to 2 ---
      genuinely lossy
\item \alert{Provenance}: which sources contributed?
      $(s, c)$ can't say --- and full provenance tracking
      is expensive (Goertzel: ``under resource constraints,
      we need heuristic compression'')
\end{itemize}

\medskip
\begin{center}
\textbf{The move}: carry \alert{sufficient statistics} as
the primary state. $(s, c)$ is a view you derive when needed.\\
For binary, this formalizes what PLN already does implicitly.\\
For richer carriers, it's genuinely new.
\end{center}

\end{frame}

%% ══════════════════════════════════════════════════════
%% SLIDE 4: The Move
%% ══════════════════════════════════════════════════════
\begin{frame}{The Move: Carry Evidence State}

Replace truth values with \alert{carried evidence state}:
\begin{itemize}
\item \textbf{State}: sufficient statistics $(n^+, n^-)$, not summaries $(s, c)$
\item \textbf{Query}: what you ask the state
\item \textbf{Extract}: answers from state --- a \alert{homomorphism}
\end{itemize}

\medskip
\begin{center}
\fbox{\begin{minipage}{0.75\textwidth}
\ttfamily\small
class WorldModel (State Query V : Type*) where\\
\quad revise \;: State -> State -> State\\
\quad empty \;\;: State\\
\quad extract : State -> Query -> V\\[4pt]
\rmfamily\itshape -- the additive regime (generic default for standard PLN):\\[2pt]
\ttfamily
class AdditiveWorldModel ... extends WorldModel where\\
\quad extract\_add : extract (W1 + W2) q\\
\quad\quad\quad\quad\quad\;\;= extract W1 q + extract W2 q
\end{minipage}}
\end{center}

\medskip
Four verbs: \alert{revise} (incorporate), \alert{query} (extract),
\alert{forget} (retract), \alert{explain} (provenance).

\end{frame}

%% SLIDE 8: The No-Go
%% ══════════════════════════════════════════════════════
\begin{frame}{The No-Go Theorem: Where Shortcuts End}

\begin{theorem}
Complete link evidence cannot, in general, be recovered
from local per-link surrogates alone.
\end{theorem}

\bigskip
This result \alert{sharpens the boundary} of what compiled
inference can achieve:

\begin{itemize}
\item Local chaining computes a useful approximation ---
      but it is not the complete semantics
\item Side conditions ($\Sigma$) make explicit \emph{when}
      the approximation is exact
\item The additive regime identifies the largest class
      where compiled shortcuts are provably correct
\end{itemize}

\medskip
Knowing the boundary is as valuable as knowing the interior.

\end{frame}

\begin{frame}{Beyond Additive: The Overlap Regime}

Not all evidence sources are independent.

\begin{description}
\item[Additive] Independent sources compose by $+$.\\
  This is the main regime --- most of the book lives here.
\item[Overlap-aware] Two sensors share a hallway vent $\Rightarrow$
  naive addition double-counts the shared signal.\\
  The overlap extraction law makes the correction explicit:
  \[
    \mathrm{extract}(\mathrm{merge}(W_1, W_2), q)
    = e_1 + e_2 - \mathrm{overlap}(W_1, W_2, q)
  \]
\end{description}

\medskip
One key result: \alert{additivity is recovered} when the shared
component is small --- so the overlap correction is needed only
where it matters.

\smallskip
{\small Further regimes (trust-gated, coalition) are defined as
typeclasses but not yet developed to comparable depth.}

\end{frame}

\begin{frame}{Why This Algebra? The KS-Evidence-Measure Connection}

Two structures, both formalized:

\medskip
\textbf{1.\ The semigroup side} (evidence revision):
\begin{itemize}
\item Evidence with $\oplus$ is an ordered commutative monoid
\item KS representation: any such monoid embeds
  \alert{uniquely} into $(\mathbb{R}, +)$ via
  $\Theta(e) = n^+ + n^-$
\item $\Theta$ then induces a measure: weight each query by
  $\Theta(\mathrm{evidence}_q)$
\end{itemize}

\medskip
\textbf{2.\ The lattice side} (evidence structure):
\begin{itemize}
\item Binary evidence is a Frame (complete Heyting algebra)
\item Dirichlet evidence inherits Frame for all $k$
\item KS valuations live on \alert{distributive lattices}
  --- Frame implies distributive, so both carriers qualify
\end{itemize}

\medskip
\alert{Additivity holds at every level}: $\oplus$ on evidence,
$+$ on reals, $+$ on measures.  The evidence quantale
(Goertzel, 2025) provides the algebraic structure;
KS provides the probabilistic grounding.

\end{frame}

\medskip
\textbf{Why it matters}:
\begin{itemize}
\item For finite-support ground programs, PLN computes the

  and supports \alert{forgetting} --- things ProbLog fundamentally cannot
\item The alarm network: same program computed 3 ways, same answer,
  but only WM-PLN can retract a sensor or explain the conclusion
\end{itemize}

\end{frame}

\begin{frame}{Demand-Directed Backward Chaining (PeTTa)}

$\text{demand}(\text{premise})
= \text{parent\_demand} \times \text{sensitivity}
\times (1 - \text{confidence})$

\medskip
\begin{center}
\footnotesize
\begin{tabular}{lccc}
\toprule
\textbf{Chainer} & \textbf{Paths} & \textbf{Expansions}
  & \textbf{Behavior} \\
\midrule
\texttt{dbc} (exhaustive) & 2 & 10 & explore all \\
\texttt{dbc-best} (demand) & 1 & 3 & highest-demand first \\
\bottomrule
\end{tabular}
\end{center}

\medskip
\textbf{Why Terrier over Collie?}
Terrier confidence $= 0.20$ (need $= 0.80$);
Collie confidence $= 0.95$ (need $= 0.05$).\\
Demand directs search to the more uncertain, more
decision-relevant premise.  \alert{70\% fewer expansions.}

\medskip
{\small\textit{Honest boundary: greedy best-first demand control.
Not yet a global agenda-based controller.}}

\end{frame}

%% ══════════════════════════════════════════════════════
\begin{frame}[fragile]{PLN Backward Chaining in PeTTa}

The chainer finds proofs, propagates truth values, and
scores premises by demand --- all in MeTTa:

\medskip
{\footnotesize
\begin{verbatim}
;; Find rule, prove premise recursively, propagate TV
(= (dbc $kb (S $d) (: ($rule $pprf) $goal))
   (let* (((proven (: $rule (-> (: $pprf $pt) $goal))
                   (stv $sr $cr))
           (dbc $kb $d (: $rule ...)))
          ((proven (: $pprf $pt) (stv $sp $cp))
           (dbc $kb $d (: $pprf $pt))))
     (proven (: ($rule $pprf) $goal)
             (stv (* $sp $sr) (* $cp $cr)))))

;; Demand = urgency × sensitivity × uncertainty
(= (demand-at $pd (stv $sr $cr) (stv $sp $cp))
   (min 1.0 (* (* $pd (max $sr $cr)) (- 1.0 $cp))))
\end{verbatim}}

\end{frame}

\begin{frame}{The Big Picture}

Meredith defines: \emph{a physics is a GSLT equipped with a
weight map} (complex amplitudes on rewrite steps) and a cost map
(resource consumption). The WM calculus is that weight map
instantiated for evidence --- every GSLT has a canonical additive
evidence algebra via multiset counting.

\bigskip
The WM calculus is therefore not specific to probabilistic logic.
It is the \alert{evidence layer for any computational system}
with parallel composition --- process calculi, concurrent
languages, MeTTa, and the $\rho$-calculus that Hyperon builds on.

\bigskip
The carrier $V$ determines what kind of reasoning you get:
\begin{itemize}
\item $V = \langle n^+, n^- \rangle$: binary PLN evidence
\item $V = (\alpha_1, \ldots, \alpha_k)$: categorical (Dirichlet)
\item $V = (n, \sum x_i, \sum x_i^2)$: continuous (Normal-Gamma)
\item $V = \mathbb{C}$: quantum amplitudes (Meredith's framework)
\end{itemize}
All formalized.  The Knuth-Skilling representation theorem
guarantees each is the \emph{unique} algebra satisfying its
axioms.

\end{frame}
